    github page with Hugo 블로그 가이드

    🚀 Hugo 블로그 자동 배포 A to Z#

    이 가이드는 Hugo 소스 코드와 배포 결과물을 분리하고, GitHub Actions로 배포를 자동화하며, Codespaces로 어디서든 글을 쓸 수 있는 환경을 구축하는 것을 목표로 합니다.

    1. 2개의 저장소(Repository) 준비#

    가장 먼저 소스 코드(원본)와 배포용(결과물)을 담을 두 개의 GitHub 저장소를 준비합니다.

    1. A. 소스 코드 저장소 (비공개 권장)

      • 목적: Hugo 원본 파일(content/, themes/, hugo.toml 등)을 관리합니다.
      • 액션: GitHub에서 my-blog-source (이름은 자유)라는 Private 저장소를 생성합니다.

    2. B. HUGOPAGE 배포용 저장소 (공개 필수)

      • 목적: Hugo가 빌드한 HTML/CSS 결과물이 올라갈 공간.
      • 액션: GitHub에서 wavewwave20.github.io 이름으로 Public 저장소를 생성합니다. (반드시 [사용자이름].github.io 형식이어야 합니다.)

    2. 배포 키(Deploy Key) 설정 (A가 B에 접근할 권한)#

    A 저장소(봇)가 B 저장소에 코드를 push할 수 있도록 SSH 키(열쇠)를 발급하고 등록합니다.

    1. SSH 키 생성 (로컬 터미널):

      ssh-keygen -t rsa -b 4096 -f hugo-deploy-key
# (비밀번호 입력 없이 Enter 두 번)
# 'hugo-deploy-key' (개인 키), 'hugo-deploy-key.pub' (공개 키) 파일 2개 생성됨

    2. B 저장소 (wavewwave20.github.io) 에 ‘공개 키’ 등록:

      • Settings > Deploy keys > Add deploy key.
      • Title: ACTIONS_DEPLOY_KEY (알아보기 쉬운 이름).
      • Key: hugo-deploy-key.pub 파일의 내용을 복사 (cat hugo-deploy-key.pub).
      • Allow write access: 반드시 체크.

    3. A 저장소 (my-blog-source) 에 ‘개인 키’ 등록:

      • Settings > Secrets and variables > Actions > New repository secret.
      • Name: ACTIONS_DEPLOY_KEY (나중에 .yml 파일에서 쓸 이름, 위 Title과 달라도 됨).
      • Secret: hugo-deploy-key (.pub 없는 개인 키) 파일 내용을 복사 (cat hugo-deploy-key).

    3. Hugo 기본 설정 및 테마 설치#

    이제 로컬이나 코드스페이스에서 A 저장소(my-blog-source)를 클론받아 Hugo 설정을 시작합니다.

    1. A 저장소 클론:

      git clone https://github.com/wavewwave20/my-blog-source.git
cd my-blog-source

    2. Hugo 사이트 초기화 (주의!):

      • 만약 my-blog-source가 완전히 빈 폴더라면: hugo new site . --force (현재 폴더에 Hugo 설치)
      • 이미 hugo new site를 한 상태라면 이 단계는 통과.

    3. PaperMod 테마 설치 (Submodule 방식 권장):

      Hugo 공식 테마 사이트에서 마음에 드는 테마를 고릅니다. (이 블로그는 PaperMode 테마를 사용함

      git submodule add https://github.com/adityatelange/hugo-PaperMod.git themes/papermod

    4. hugo.toml (또는 config.toml) 기본 설정:

      • hugo.toml 파일을 열어 최소한 다음 내용을 설정합니다.
      baseURL = "https://wavewwave20.github.io/"
languageCode = "ko-kr"
title = "[블로그 이름]"
theme = "[테마이름]"

# (PaperMod 테마에 필요한 추가 설정들... 예: menu, params 등)

    4. GitHub Actions 워크플로우 설정#

    A 저장소에 배포 자동화 스크립트(.yml)를 추가합니다.

    1. 폴더 및 파일 생성:

      • A 저장소(my-blog-source) 루트에 .github/workflows/ 폴더를 만듭니다.
      • 그 안에 deploy.yml 파일을 생성합니다.

    2. deploy.yml 코드 작성:

      name: GitHub Pages Deploy
on:
  push:
    branches: [ "main" ] # main 브랜치에 push 될 때 실행

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: 'recursive'  # 1. 테마(submodule) 가져오기 (필수!)

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'
          extended: true          # 2. PaperMod는 extended 버전 필수!

      - name: Build
        run: hugo --minify      # 3. hugo 빌드 (public 폴더 생성)

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v4
        with:
          deploy_key: ${{ secrets.ACTIONS_DEPLOY_KEY }}     # 4. 2단계에서 등록한 Secret 키
          external_repository: wavewwave20/wavewwave20.github.io  # 5. B 저장소 경로
          publish_dir: ./public                         # 6. 빌드된 public 폴더
          publish_branch: main                          # 7. B 저장소의 main 브랜치로 푸시

    5. Codespaces 개발 환경 세팅#

    A 저장소에서 코드스페이스를 열 때마다 Hugo를 자동 설치하도록 설정합니다.

    1. 폴더 및 파일 생성:

      • A 저장소 루트에 .devcontainer/ 폴더를 만듭니다.
      • 그 안에 devcontainer.json 파일을 생성합니다.

    2. devcontainer.json 코드 작성:

      {
  "name": "Hugo Dev",
  "image": "mcr.microsoft.com/devcontainers/base:ubuntu",
  "features": {
    // Hugo (extended 버전) 자동 설치
    "ghcr.io/devcontainers/features/hugo:1": {
      "version": "latest",
      "extended": true
    }
  },
  // Hugo 서버 포트(1313) 자동 개방
  "forwardPorts": [1313]
}

    6. 첫 글 작성 및 배포 (테스트)#

    이제 모든 설정이 완료되었습니다. 이 단계부터는 앞으로 계속 반복할 글쓰기 루틴입니다.

    1. (로컬 또는 Codespaces 터미널에서) A 저장소로 이동.

    2. hugo.toml, themes/ 등 모든 설정 파일 push (최초 1회):

      • 3~5단계에서 작업한 모든 파일을 A 저장소(my-blog-source)에 push 합니다.
      git add .
git commit -m "최초 Hugo 및 자동화 설정"
git push origin main
      • (A 저장소의 Actions 탭에서 deploy.yml이 실행되어 B 저장소에 빈 사이트가 배포되는지 확인)

    3. 새 글 생성:

      hugo new posts/my-first-post.md

    4. 미리보기 (선택 사항이지만 권장):

      hugo server -D
# (Codespaces의 경우 '포트' 탭에서 1313 포트의 URL 클릭)

    5. 글 작성:

      • content/posts/my-first-post.md 파일을 열어 내용을 작성합니다.
      • 작성이 완료되면 상단의 draft: true를 draft: false로 변경하거나 삭제합니다.

    6. 커밋 및 푸시 (배포 트리거):

      • 이것이 최종 배포 단계입니다.
      git add .
git commit -m "첫 번째 포스트 작성 완료"
git push origin main

    7. 최종 확인#

    1. A 저장소(my-blog-source)의 [Actions] 탭으로 이동합니다.
    2. 방금 push한 “첫 번째 포스트 작성 완료” 커밋에 대한 워크플로우가 **초록색 체크 ✔**로 완료되었는지 확인합니다.
    3. 웹 브라우저에서 https://wavewwave20.github.io/에 접속하여 새 글이 정상적으로 보이는지 확인합니다. (캐시 문제 시 Ctrl+Shift+R로 강력 새로고침)

    부록#

    1. hugo 명령어#

    명령어설명
    hugo new [섹션]/[파일이름].md새 글을 생성하는 가장 기본 명령어입니다.
    hugo server로컬 서버를 실행합니다. (초안 글 안 보임)
    hugo server -D(가장 자주 씀) 초안(draft) 글까지 포함하여 로컬 서버를 실행합니다.
    hugo사이트를 public 폴더로 빌드(배포용 HTML 생성)합니다.

    2. 현재시간 hugo date 형식으로 cli 출력#

    get_time.sh 파일을 만들고 다음 내용을 복사

    #!/bin/bash

# 시간대를 KST(Asia/Seoul)로 강제 지정합니다.
export TZ="Asia/Seoul"

# ISO 8601 형식 (YYYY-MM-DDTHH:MM:SS+09:00)으로 날짜를 출력합니다.
# %:z는 시간대에 콜론(:)을 포함시킵니다.
date +"%Y-%m-%dT%H:%M:%S%:z"

    2.파일에 실행 권한을 줍니다.

    chmod +x get_time.sh

    3.파일을 실행합니다.

    ./get_time.sh
